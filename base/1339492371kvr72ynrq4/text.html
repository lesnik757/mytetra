проверить и исправить ошибки файловой системы EXT2 на жестком диске, который подключен по eSata к Egreat EG-M31B? Telnet на плеере установлен, запускаю его на компе, ввожу команду "open 192.168.1.7", получаю следующее:

/opt/sybhttpd/localhost.drives/HARD_DISK #


Как мне перейти на диск с названием "USB_DRIVE_A-1", который содержит всего один раздел в EXT2, проверить его на ошибки и при необходимости исправить?


1. С ПК, через телнет C:\WINDOWS\system32\cmd.exe: telnet 192.168.1.xx (xx четвертый октет медиатанка в лок сети)

login: root без логина

без пароля


2. # fdisk -l


Получаем инфо что-то вроде:


Disk /dev/sdc: 1026 MB, 1026555904 bytes

255 heads, 63 sectors/track, 124 cylinders

Units = cylinders of 16065 * 512 = 8225280 bytes


Device Boot Start End Blocks Id System

/dev/sdc1 * 1 124 995998+ 83 Linux



3. Вводим команду чекинга ошибок # e2fsck -f /dev/sdc1


Система плеера пишет:


e2fsck 1.38 (30-Jun-2005)

/dev/sdc1 is mounted.


WARNING!!! Running e2fsck on a mounted filesystem may cause

SEVERE filesystem damage.


Сообщает, что начнет процесс исправления ошибок, с изменением файловой системы, это нормально! Соглашаемся.
